# Design Handoff

구현 전에 화면 구조와 상태를 고정하는 스킬이다.

## 입력

- 사용자 목표와 대상 사용자
- 기존 브랜드 또는 참고 화면
- 저장소의 현재 화면 구조
- `_workspace/00_request_snapshot.md`

## 출력

- `_workspace/01_design_brief.md`
- `_workspace/02_ui_contract.md`
- 필요하면 `_workspace/02_copy_notes.md`

## 절차

1. 사용자가 실제로 거치는 핵심 흐름을 1~3개로 압축한다.
2. 각 흐름마다 필요한 페이지나 섹션을 정리한다.
3. 페이지별로 다음을 문서화한다.
   - 목표
   - 핵심 컴포넌트
   - 필요한 데이터
   - 빈 상태
   - 로딩 상태
   - 오류 상태
   - 모바일/데스크톱 차이
4. `_workspace/02_ui_contract.md`에는 구현자가 바로 쓸 수 있게 다음 형식으로 적는다.
   - 화면 또는 라우트
   - 컴포넌트 목록
   - 각 컴포넌트의 props 수준 책임
   - 사용자 액션
   - 기대 응답
5. 백엔드 의존 데이터는 endpoint 후보 또는 데이터 소스로 함께 적는다.

## 품질 기준

- 프론트엔드가 디자인 해석으로 헤매지 않아야 한다.
- 백엔드가 필요한 데이터를 추측하지 않아야 한다.
- 시각적 방향보다 상태 정의가 빠지지 않는 것이 더 중요하다.
