# 에코 체크: 우리 교실 탄소 발자국

## 1. MVP 목표
- 초등 4~6학년 학생이 교실에서 발생하는 쓰레기와 전기 사용량을 쉽게 입력할 수 있다.
- 입력한 값이 즉시 대시보드 숫자와 막대 그래프에 반영된다.
- 오늘 우리 교실의 탄소 배출 상태를 직관적으로 확인할 수 있다.

## 2. MVP 범위
### 포함
- 쓰레기 종류별 수량 입력
- 전기 사용 시간 입력
- 총합 대시보드
- 항목별 막대 그래프
- 간단한 탄소 환산 결과
- 전체 초기화
- 새로고침 후 데이터 유지(localStorage)

### 제외
- 로그인
- 서버/DB 저장
- 학급별 비교
- 날짜별 히스토리
- 교사용 관리자 페이지

## 3. 사용자 시나리오
1. 학생이 웹 앱을 연다.
2. 오늘 버린 쓰레기나 사용한 전기 시간을 항목별로 입력한다.
3. 입력 직후 총 수치와 그래프가 바뀐다.
4. 앱이 오늘의 총 탄소 배출량과 가장 많은 항목을 보여준다.
5. 활동이 끝나면 초기화하거나 그대로 저장해 둘 수 있다.

## 4. MVP 화면 구조
### 전체 레이아웃
- 상단 헤더
- 메인 2단 영역
- 하단 안내 영역

### 화면 와이어프레임
```text
+------------------------------------------------------+
| 에코 체크: 우리 교실 탄소 발자국                      |
| 오늘 날짜 / 학습 목표                                 |
+---------------------------+--------------------------+
| 입력 패널                 | 오늘의 대시보드          |
|                           |                          |
| [플라스틱] [-]  3  [+]    | 총 입력 수량: 18         |
| [종이]     [-]  5  [+]    | 예상 탄소 배출: 2.4kg    |
| [캔]       [-]  1  [+]    | 가장 많은 항목: 종이     |
| [일반]     [-]  4  [+]    | 에코 상태: 보통          |
| [전기사용] [-] 30 [+]분   |                          |
|                           |                          |
| [전체 초기화]             |                          |
+------------------------------------------------------+
| 항목별 그래프 (Chart.js 막대 그래프)                 |
+------------------------------------------------------+
| 오늘의 한마디 / 실천 팁                               |
+------------------------------------------------------+
```

## 5. 화면 섹션 상세
### 5.1 헤더
- 앱 제목
- 오늘 날짜
- 수업 연계 문구
  - 예: "환경을 지키는 생활 습관을 데이터로 확인해요"

### 5.2 입력 패널
- 항목 목록
  - 플라스틱
  - 종이
  - 캔
  - 일반쓰레기
  - 전기사용(분)
- 입력 방식
  - `-` 버튼
  - 현재 값 표시
  - `+` 버튼
- 보조 규칙
  - 0 아래로 내려가지 않음
  - 전기사용은 10분 단위 증감으로 단순화 가능

### 5.3 대시보드 카드
- 총 입력 수량
- 예상 탄소 배출량
- 가장 많이 나온 항목
- 에코 상태
  - 좋음
  - 보통
  - 주의

### 5.4 그래프 영역
- Chart.js 막대 그래프 1개
- X축: 항목 이름
- Y축: 입력값 또는 환산값
- 색상은 항목별 고정

### 5.5 하단 안내 영역
- 오늘의 실천 팁 1개 표시
- 예: "종이를 아껴 쓰면 탄소 배출을 줄일 수 있어요"

## 7. 핵심 기능 구조
### 입력 처리
- 버튼 클릭 시 항목 값 증감
- 변경 후 상태 저장
- 화면과 그래프 즉시 갱신

### 계산 처리
- 총 입력 수량 계산
- 탄소 총합 계산
- 최다 항목 계산
- 상태 메시지 계산

### 저장 처리
- 상태를 localStorage에 저장
- 앱 시작 시 저장된 값 복원

## 8. 파일 구성안
```text
carbon-footprint/
├─ index.html
├─ styles.css
├─ app.js
├─ data.js
├─ chart.js
├─ storage.js
└─ assets/
   └─ icons/
```

## 9. 파일별 역할
### `index.html`
- 전체 레이아웃 마크업
- 헤더, 입력 패널, 대시보드, 차트 캔버스 배치
- Chart.js CDN 연결

### `styles.css`
- 전체 레이아웃
- 카드 스타일
- 버튼 스타일
- 초등학생 친화형 큰 버튼과 높은 가독성 색상 적용
- 반응형 처리

### `app.js`
- 앱 시작점
- 상태 초기화
- 이벤트 바인딩
- 전체 렌더 흐름 연결

### `data.js`
- 항목 메타 정보
- 탄소 계수
- 상태 메시지 기준값

### `chart.js`
- Chart.js 인스턴스 생성
- 차트 데이터 갱신 함수 제공

### `storage.js`
- localStorage 저장/불러오기 함수 분리

### `assets/icons/`
- 쓰레기/전기 아이콘 저장
- MVP에서는 없어도 되지만 확장 대비용 디렉터리

## 10. 구현 우선순위
1. `index.html`로 화면 뼈대 작성
2. `styles.css`로 기본 레이아웃 완성
3. `data.js`에 항목 정의
4. `app.js`에 상태 관리와 계산 로직 구현
5. `chart.js`에 막대 그래프 연결
6. `storage.js`로 localStorage 저장 추가
7. 문구와 색상 다듬기

## 11. MVP 완료 기준
- 모든 항목이 버튼으로 입력된다.
- 입력 직후 대시보드 수치가 바뀐다.
- 그래프가 실시간으로 갱신된다.
- 탄소 총합이 계산되어 표시된다.
- 새로고침 후에도 값이 유지된다.
- 초기화 버튼이 전체 상태를 정상적으로 리셋한다.
